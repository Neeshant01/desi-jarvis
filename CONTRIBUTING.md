# Contributing

Thanks for helping improve Desi Jarvis.

## Good contribution areas

- documentation clarity
- safer automation behaviors
- code readability and modularity
- cross-platform support
- better examples for plugins and configuration

## Before opening a pull request

1. Keep the project beginner-friendly.
2. Avoid unsafe or destructive automation defaults.
3. Test the affected command path when possible.
4. Document config changes clearly.

## Pull request expectations

- explain the user value
- mention environment assumptions
- describe any manual testing performed
